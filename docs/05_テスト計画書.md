## 1. テスト方針

### 1.1 基本方針

- **テストピラミッド** を遵守：単体多め、結合中程度、E2E は主要ハッピーパス中心
- **コア機能はテストカバレッジ 80% 以上**（目標 G-04）
- 純粋関数（日時処理・バリデーション・ドメインロジック）は 100% 単体テスト対象
- Google API 呼び出しは MSW（Mock Service Worker）で擬似化
- 認証フローおよび実 API 連携は手動テストでカバー

### 1.2 テストレベル

```
       ┌──────────┐
       │   E2E    │  Playwright（10〜15 シナリオ）
       │  (UI)    │
      ┌┴──────────┴┐
      │   結合     │ Testing Library + MSW（主要画面）
      │ (Feature)  │
    ┌─┴────────────┴─┐
    │     単体        │ Vitest（純粋関数・hooks）
    │  (Unit)         │
    └─────────────────┘
```

## 2. テストレベル詳細

### 2.1 単体テスト（Unit）

| 対象 | テスト内容 | フレームワーク |
|------|----------|--------------|
| 日時ユーティリティ (`lib/time/jst.ts`) | JST/UTC変換、月初・月末判定、曜日計算 | Vitest |
| バリデータ (`features/*/validators.ts`) | Zod スキーマの境界値テスト | Vitest |
| シリアライザ (`lib/google/serializers.ts`) | 行↔オブジェクト変換、ヘッダー欠落時の例外 | Vitest |
| ルーチン評価 (`features/routines/scheduleEvaluator.ts`) | 全 Schedule 種別の判定 | Vitest |
| Calendar 色マッピング | Status → colorId | Vitest |

### 2.2 結合テスト（Integration）

| 対象 | テスト内容 | フレームワーク |
|------|----------|--------------|
| useTasks フック | キャッシュ動作、楽観的更新、ロールバック | Testing Library + MSW |
| AddTaskForm | バリデーション、送信、エラー表示 | Testing Library |
| TodayRoute | 進行中・次・一覧の3要素が正しく描画 | Testing Library + MSW |
| 同期エンジン (`features/sync`) | カレンダー差分検出、削除検出 | Vitest + モックAPI |

### 2.3 E2E テスト（End-to-End）

Playwright で実行。本番ではなくモックバックエンド（MSW + 静的フィクスチャ）で実行する。

| シナリオID | シナリオ |
|----------|---------|
| E2E-01 | ログインからタスク追加までの一連の流れ |
| E2E-02 | タスク開始 → タイマー作動 → 終了 |
| E2E-03 | 確認待ちタスクの追加と完了 |
| E2E-04 | ルーチンタスク翌週分の生成 |
| E2E-05 | オフライン状態での閲覧と操作キューイング |
| E2E-06 | 再オンライン時の自動同期 |
| E2E-07 | 認証エラー発生時のリログインフロー |
| E2E-08 | バリデーションエラーの表示 |
| E2E-09 | カテゴリ追加・選択 |
| E2E-10 | モバイル表示でのボトムタブ操作 |

### 2.4 手動テスト

CI で自動化困難な以下は手動で実施。

| 項目 | 内容 |
|------|------|
| OAuth 同意フロー | 実際の Google アカウントで同意取得 |
| PWA インストール | iOS Safari / Android Chrome でホーム追加 |
| 通知 | 見積超過時の OS 通知が出る |
| 実 API 連携 | 本物の Sheets / Calendar / Tasks への CRUD |
| 表示確認 | PC・iPhone・Android で各画面のレイアウト |

## 3. 主要テストケース（シナリオベース）

### TC-01: タスク追加（正常系）

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | タスク追加画面でタスク名「テスト」、見積30と入力 | フォームに値が保持される |
| 2 | カテゴリ「管理」を選択 | プルダウンが「管理」を表示 |
| 3 | 「追加」ボタンを押す | 200ms 以内にトースト「タスクを追加しました」表示 |
| 4 | 今日タブに移動 | 追加されたタスクが一覧末尾に表示 |
| 5 | Calendar を開く | 対応するイベントがGRAY色で作成されている |
| 6 | TaskDB シートを開く | 行が追加され、Status=Not Started |

### TC-02: タスク開始・終了（正常系）

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 次のタスクがある状態で「▶ 次を開始」を押す | ステータスが In Progress に変更 |
| 2 | タイマーが起動する | 1秒ごとに表示が更新される |
| 3 | 見積の50%経過 | 進捗バーが50%、オレンジ色 |
| 4 | 見積を超過する | 進捗バーが赤色、超過分の分数表示 |
| 5 | 「■ 現在を終了」を押す | Status=Done に変更、タイマー停止 |
| 6 | Calendar を確認 | イベントがGREEN色、実績時刻にリサイズ |

### TC-03: バリデーション

| # | 入力 | 期待結果 |
|---|------|---------|
| 1 | タスク名を空のまま「追加」 | 「タスク名を入力してください」と表示、API呼び出しなし |
| 2 | 見積に -10 を入力 | 「見積は1〜480分の範囲で入力してください」 |
| 3 | 見積に 999 を入力 | 同上 |
| 4 | 確認待ちのフォローアップ日に過去日 | 「過去の日付は指定できません」 |

### TC-04: オフライン動作

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | オンライン状態でアプリを起動し、データを読み込む | IndexedDB にミラー保存される |
| 2 | DevTools でオフラインに設定 | 既存タスクは閲覧可能 |
| 3 | オフラインで「次を開始」を押す | UI上は In Progress に変更、トースト「オフライン中の操作を保存」 |
| 4 | オンラインに復帰 | 30秒以内に同期完了、Calendar も更新される |
| 5 | コンソールに `mutation_queue` の処理ログ | エラーなく完了 |

### TC-05: 認証エラー時の復旧

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | アクセストークンを手動で無効化 | API 呼び出しが 401 を返す |
| 2 | アプリが自動 silent renewal を試行 | 成功すれば処理続行 |
| 3 | silent renewal も失敗 | モーダル「再ログインが必要です」表示 |
| 4 | 「ログイン」を押す | OAuth ポップアップ表示 |
| 5 | 同意完了 | 直前の操作が自動再実行される |

### TC-06: ルーチンタスク生成

| # | 前提・操作 | 期待結果 |
|---|----------|---------|
| 1 | RoutineTasks に `毎日`/`月`/`末日` の3件を定義 | - |
| 2 | 「翌週のルーチンタスクを生成」を押す | 月〜金分が評価され、該当タスクのみ生成 |
| 3 | 同じボタンをもう一度押す | 重複追加されない（既存とのキー比較で除外） |
| 4 | Calendar とシート両方を確認 | イベントと行が同数存在 |

### TC-07: 確認待ちタスクの双方向同期

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | アプリから確認待ちタスクを追加 | Google ToDo にも `[WAIT] ...` で作成 |
| 2 | Google ToDo 側でタイトルを編集 | アプリの WaitingList も更新される |
| 3 | Google ToDo 側で完了 | アプリ側で取り消し線表示 |
| 4 | Google ToDo 側で削除 | アプリの WaitingList から行も削除（インデックスズレなし） |

### TC-08: タイムゾーン境界

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | フォローアップ日 2026-05-19（JST）を選択 | Google ToDo の `due` が 2026-05-19 として正しく表示 |
| 2 | アプリで再表示 | 入力した 2026-05-19 がそのまま表示される（UTC変換で前日にならない） |

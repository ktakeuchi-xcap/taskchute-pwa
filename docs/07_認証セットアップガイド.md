# OAuth セットアップガイド

| 項目     | 内容                                                         |
| -------- | ------------------------------------------------------------ |
| 対象     | Taskchute PWA を起動する開発者                               |
| 所要時間 | 約 10 分                                                     |
| 前提     | Google Workspace アカウント、Google Cloud Console アクセス権 |

このドキュメントは M2（認証実装フェーズ）の前に 1 回だけ実施すれば OK。

---

## 1. 全体像

PWA からは **Google Identity Services (GIS)** を経由して Google API を叩く。SPA のため OAuth クライアント シークレットは使えず、**Web アプリケーション** タイプのクライアント ID を発行して、その ID だけをフロントに埋め込む（公開しても安全）。

```
┌──────────┐   1. ログイン要求   ┌──────────────┐
│ Taskchute │ ─────────────────▶ │  Google GIS  │
│   PWA     │ ◀───────────────── │              │
└──────────┘   2. access_token   └──────────────┘
       │
       │ 3. Bearer token で API 呼び出し
       ▼
┌────────────────────────────────────────┐
│ Sheets / Calendar / Tasks / userinfo  │
└────────────────────────────────────────┘
```

---

## 2. Google Cloud Console での作業

### 2.1 プロジェクトを作成（既存利用でも可）

1. <https://console.cloud.google.com/> にアクセス
2. プロジェクト選択メニュー →「新しいプロジェクト」
3. プロジェクト名：`Taskchute PWA`（任意）

### 2.2 必要な API を有効化

「API とサービス」→「ライブラリ」で以下を有効化：

- **Google Sheets API**
- **Google Calendar API**
- **Google Tasks API**

`People API` は不要（userinfo エンドポイントは OAuth2 v3 の標準として常に利用可）。

### 2.3 OAuth 同意画面を構成

「API とサービス」→「OAuth 同意画面」：

- ユーザータイプ：**内部**（Workspace 組織内のみ）
- アプリ名：`Taskchute PWA`
- ユーザーサポート メール：`k.takeuchi@xcap.co.jp`
- 承認済みドメイン：`github.io`（GitHub Pages デプロイ用）、ローカルテスト時のトンネルドメインは個別追加不要

スコープの追加：

| スコープ                    | 用途                |
| --------------------------- | ------------------- |
| `openid`                    | OpenID Connect      |
| `.../auth/userinfo.email`   | ユーザーメール表示  |
| `.../auth/userinfo.profile` | プロフィール表示    |
| `.../auth/spreadsheets`     | TaskDB ほか読み書き |
| `.../auth/calendar.events`  | カレンダーイベント  |
| `.../auth/tasks`            | Google ToDo         |

テストユーザー（外部公開時のみ）：`k.takeuchi@xcap.co.jp`

### 2.4 認証情報の作成

「API とサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアント ID」：

- アプリケーションの種類：**ウェブ アプリケーション**
- 名前：`Taskchute PWA Web Client`
- **承認済みの JavaScript 生成元**：
  - `http://localhost:5173`
  - `https://<github-username>.github.io`
  - （Cloudflare Tunnel を使う場合は発行された URL を追加）
- **承認済みのリダイレクト URI**：登録不要（GIS は postMessage で完結する）

「作成」を押すとクライアント ID が発行される。形式は `1234567890-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.apps.googleusercontent.com`。

## 3. 環境変数の設定

プロジェクトルートに `.env.local` を作成（git 管理外）：

```bash
cp .env.example .env.local
```

`.env.local` を編集：

```dotenv
VITE_GOOGLE_OAUTH_CLIENT_ID=1234567890-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.apps.googleusercontent.com
VITE_TASKCHUTE_SPREADSHEET_ID=1abcDEF...（既存 Taskchute スプレッドシートの ID）
VITE_TASKCHUTE_CALENDAR_ID=c_xxxxxxxxxxxxxxx@group.calendar.google.com
```

> **スプレッドシート ID の見つけ方**: `https://docs.google.com/spreadsheets/d/<ここがID>/edit` の `<ここがID>` 部分。

> **カレンダー ID の見つけ方**: Google Calendar の対象カレンダーの「設定」→「カレンダーの統合」→「カレンダー ID」。

## 4. 動作確認

```bash
npm run dev
```

ブラウザで <http://localhost:5173/> を開くと、未認証の場合はログイン画面が表示される。「Google でサインイン」を押下し、同意画面で許可するとアプリ本体が表示される。

### スマホからの確認

OAuth は HTTPS 必須のため、LAN 直アクセス（`npm run dev:host`）では機能しない。**Cloudflare Tunnel** 経由でテストする：

```bash
brew install cloudflared      # 初回のみ
npm run dev                    # Terminal A
npm run dev:tunnel             # Terminal B → https://xxx.trycloudflare.com が発行される
```

その URL を **「承認済みの JavaScript 生成元」に追加** してから、スマホでアクセス。

> Quick Tunnel は起動のたびに URL が変わるため、頻繁にスマホ実機テストする場合は GitHub Pages にデプロイ済みの URL を使う方が楽。

## 5. トラブルシューティング

| 症状                                                 | 原因                                   | 対処                                                              |
| ---------------------------------------------------- | -------------------------------------- | ----------------------------------------------------------------- |
| ログイン画面で「OAuth クライアントIDが未設定」と出る | `.env.local` 未設定 or 変数名違い      | `VITE_GOOGLE_OAUTH_CLIENT_ID` を確認                              |
| `idpiframe_initialization_failed`                    | 生成元 URL が承認済みリストに無い      | Google Cloud Console で URL を追加（5 分程度反映待ち）            |
| `popup_closed_by_user`                               | ユーザーがポップアップを閉じた         | 想定挙動。再度サインインボタンを押す                              |
| `access_denied`                                      | 同意画面で「キャンセル」               | 必要スコープの説明を確認                                          |
| サイレント更新が常に失敗する                         | サードパーティ Cookie ブロック・ITP 等 | Safari の場合：トラッキング防止を一時 OFF、または通常モードで利用 |
| 同意画面で「このアプリは確認されていません」         | 同意画面が「テスト」状態               | 開発中は問題なし。本番化時は「公開」へ                            |

---

## 6. セキュリティ上の注意

- **クライアント ID は公開して問題ない**：SPA の構造上、ビルド成果物に含まれる
- **アクセストークン**は sessionStorage 限定（localStorage には保管しない）。タブを閉じれば破棄される
- **スコープは最小限**：今後機能追加で他スコープが必要になっても、必要になった段階で追加する
- **承認済み生成元**には実際に使う URL のみ登録（古い trycloudflare URL は定期的に削除）
